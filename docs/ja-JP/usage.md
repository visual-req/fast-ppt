# 使い方

## 利用シーン

ユーザーが入力資料を PPT / deck / プレゼンテーションに変換したい場合に、この Skill を使用します。

典型的な依頼：
- 「PPT を生成して」
- 「この資料を deck にして」
- 「ブラウザで見られる Web PPT を出力して」

## 基本フロー

### 1. アウトライン生成

`/fppt:outline` を使用：

- PPT タイプの判定
- 章の計画
- ページ意図とレイアウトの選択
- `outline.json` の出力

### 2. アウトライン確認

詳細ページ生成前に以下を確認：

- 章順序が適切か
- ページ意図が妥当か
- レイアウト選択が適切か
- キーページの欠落がないか

### 3. 詳細ページ生成

`/fppt:detail` を使用：

- `deck.json` 生成
- `slides/*.json` 生成
- `work/assets/*.svg` 生成または参照
- 図表・構造図・比較表・フェーズ図の補完

### 4. QA

2 ラウンドの QA：

- アウトライン QA
- 詳細ページ QA

各ラウンドで最低限：通過項目・未通過項目・修正アクション・フェーズ判定を出力。

## 出力ディレクトリ

```
work/ppt/001_プロジェクト/
  outline.json
  deck.json
  slides/
```

アセット出力先：

```
work/assets/
```
