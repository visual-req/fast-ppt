# `/fppt:review` — 听众视角审阅

对已生成的 `deck.json + slides/*.json`，从听众视角进行多维度审阅，输出 `work/ppt/XXX/review-yyyyMMddHHmmss.md`。

## 流程

1. 读取 `deck.json` 和所有 `slides/*.json`
2. 根据 `deck.json` 中的 style 字段判断 PPT 类型（课件类/方案类/汇报类/演示类）
3. 先检查整套 deck 的页顺序、章节划分、前后承接，再按对应类型的审阅维度逐页检查
4. 输出结构化审阅报告到项目目录下的 `review-yyyyMMddHHmmss.md`

## 审阅立场

你不是作者，你是一个第一次看到这份 PPT 的听众。你的核心问题是：

- 我能不能在 30 秒内理解这一页在说什么？
- 这一页的说服力够不够——不论是用数据、案例还是逻辑？
- 相邻页面之间的逻辑是否连贯？
- 信息密度是否合适（太少废话，太多记不住）？
- 这页是不是出现在正确的章节、正确的顺序上？
- 页面内容、标题、主题三者是否真的一致？

## 通用审阅维度（所有类型共享）

对每一页，从以下维度打分（1-5），并给出具体修改建议：

| 维度 | 说明 | 1分 | 5分 |
|------|------|-----|-----|
| 可理解性 | 30秒内能否抓住要点 | 完全看不懂 | 一眼明了 |
| 结构清晰度 | 信息组织是否有层次 | 杂乱无章 | 逻辑严密 |
| 信息密度 | 内容量是否恰当 | 空洞/过载 | 恰到好处 |
| 视觉匹配度 | layout_type 是否适合内容 | 完全不适合 | 最佳匹配 |
| 承接连贯性 | 与前后页的逻辑衔接 | 跳跃断裂 | 自然过渡 |
| 章节归属合理性 | 页面是否放在正确章节 | 明显放错 | 归属准确 |
| 主题关联性 | 标题、内容、章节主题是否一致 | 明显跑题 | 高度一致 |

## 结构与归属专项检查

在逐页审阅之前，必须先做一轮整套 deck 结构检查：

1. 页顺序是否合理：是否存在业务页误放到研发章节、办公页误放到业务章节、总结页早于论证页等问题。
2. 章节归属是否合理：页面内容是否与当前 `section_divider` 或章节主题相匹配。
3. 前后承接是否合理：上一页提出的问题，下一页是否真的回答；是否出现跳跃、重复、倒置。
4. 重复与缺口：是否有重复表达同一信息的页面，或某个章节缺少关键承接页。
5. 文件同步检查：`deck.json` 与 `outline.json` 的 slide 数量、编号、顺序是否一致；`slides/` 目录下的文件名是否从 001 开始连续编号无跳号。

## 页面内容专项检查

每一页除常规评分外，还必须补充以下判断：

1. 标题与内容是否一致：标题不能讲 A，正文却在讲 B。
2. 内容与章节主题是否一致：例如“办公 AI”内容不能放在“软件研发 AI”章节，“运维可观测”不能放在业务办公章节。
3. 布局选择是否合理：步骤页要有顺序推进；九宫格要真的是并列能力地图；表格页要适合审阅比对；`svg_full` 要承担主表达而非仅作装饰。
4. 内容密度是否匹配布局：如果文本太长导致卡片塞满、图形过小、表格难读，应判为布局不合理。

## 图形页专项检查

当页面包含 `svg_full`、图表、流程图、阶段图、甘特图、雷达图等图形表达时，必须额外检查：

1. 图的结构是否完整：是否有缺框、缺节点、缺连线、缺图例、缺坐标、缺刻度。
2. 图的内容是否正确：图内文案是否与页标题和章节主题一致，是否存在语义错位。
3. 图的序号与步骤是否正确：阶段编号、步骤编号、箭头方向、顺序是否前后一致。
4. 图的对齐与排版是否稳定：节点是否对齐、间距是否一致、文字是否压线或越界。
5. 图的可见性是否合格：在 16:9 画布中主体是否完整可见，是否依赖浏览器裁切才能成立。
6. 图文分工是否清晰：页标题承担主题，SVG 内不要重复一遍标题；图注要补充口径而非重复标题。

## 课件类审阅方案

课件类 PPT 的目标是让学员学会某个知识或技能。

### 课件类额外审阅维度

| 维度 | 评委视角 | 检查要点 |
|------|----------|----------|
| 教学目标明确度 | "学完这一页我知道自己会了什么" | 每页是否有明确的知识点或技能目标 |
| 循序渐进性 | "前面讲的后面用得上" | 概念→原理→方法→示例→练习的顺序是否合理 |
| 案例辅助性 | "例子能帮我理解抽象概念" | 关键概念是否有案例支撑，案例是否贴近实际 |
| 互动引导性 | "我不是被动看，我被引导思考" | 是否有设问、思考题或练习提示 |
| 记忆留存度 | "看完了我能记住要点" | 是否有总结、口诀、框架图帮助记忆 |

### 课件类审阅报告格式

```
## 课件类审阅报告

### 总体评价
- 教学目标达成度：X/5
- 内容循序渐进性：X/5
- 案例支撑充分性：X/5
- 互动引导设计：X/5

### 逐页审阅
#### 001_cover: 封面
- 可理解性：X/5 — 问题 + 建议
- ...

### 改进优先级
1. [高] 第X页：原因 + 建议
2. [中] ...
3. [低] ...
```

## 方案类审阅方案

方案类 PPT 的目标是说服决策者批准或采纳方案。

### 方案类额外审阅维度

| 维度 | 评委视角 | 检查要点 |
|------|----------|----------|
| 逻辑说服力 | "为什么必须做"讲透了没有 | 问题定义→方案推演→收益论证的因果链是否完整 |
| 数据说服力 | "数字让我信服" | KPI/收益/成本是否有具体数字和测算前提 |
| 案例说服力 | "别人做成了我就能放心" | 案例是否真实、可复现、有量化效果 |
| 风险诚实度 | "风险讲清楚了不是画饼" | 是否坦诚讨论风险、局限和应对措施 |
| 决策推动力 | "看完知道该拍什么板" | 是否有明确的建议动作和下一步清单 |

### 方案类审阅报告格式

```
## 方案类审阅报告

### 总体评价
- 逻辑说服力：X/5 — 问题→方案→收益链
- 数据说服力：X/5 — 量化支撑
- 案例说服力：X/5 — 真实可复现
- 风险诚实度：X/5 — 不回避风险
- 决策推动力：X/5 — 行动建议清晰

### 关键页审阅（重点检查以下页面类型）
#### cover: 标题是否精准传达方案价值
#### problem_statement: 问题是否讲透（现象/影响/根因）
#### 方案对比/架构页: 是否清晰展示方案差异化优势
#### kpi_cards/收益页: 数字是否有测算前提和口径
#### case_study: 案例是否具体可感
#### roadmap/实施页: 路径是否可执行
#### summary: 是否给出明确的决策建议

### 致命缺陷（如有）
- 听众可能提出的质疑 + 应对建议

### 改进优先级
1. [高] ...
```

## 汇报类审阅方案

汇报类 PPT 的目标是让上级快速掌握进展、问题和需要的支持。

### 汇报类额外审阅维度

| 维度 | 评委视角 | 检查要点 |
|------|----------|----------|
| 摘要效率 | "第一页就让我知道整体情况" | 执行摘要是否在一页内概括核心结论 |
| 数据可视化 | "图表让我秒懂趋势" | 图表类型是否合适、标注是否清晰、对比是否突出 |
| 问题透明度 | "问题没有藏着掖着" | 是否坦诚暴露问题、分析根因、给出改进计划 |
| 资源诉求明确度 | "知道你需要我做什么" | 是否清晰说明需要的资源、支持和决策 |
| 节奏掌控力 | "不冗长，每页都有信息量" | 页数是否精炼、无重复、无冗余 |

### 汇报类审阅报告格式

```
## 汇报类审阅报告

### 总体评价
- 摘要效率：X/5
- 数据可视化：X/5
- 问题透明度：X/5
- 资源诉求清晰度：X/5
- 节奏掌控力：X/5

### 逐页审阅
...

### 汇报"一页纸"测试
能否把核心内容压缩成1页仍不失关键信息？如不能，指出缺失是什么。

### 改进优先级
1. [高] ...
```

## 演示类审阅方案

演示类 PPT 的目标是让观众产生兴趣、理解产品/服务价值并留下深刻印象。

### 演示类额外审阅维度

| 维度 | 评委视角 | 检查要点 |
|------|----------|----------|
| 开场吸引力 | "第一页就让我想继续看" | 封面和开场页是否制造好奇或共鸣 |
| 价值主张清晰度 | "一句话能说清这是什么、解决什么问题" | 核心价值是否在开场 3 页内清晰传达 |
| 场景共鸣度 | "这就是我遇到的问题" | 场景/痛点描述是否让目标受众产生代入感 |
| 差异性展示 | "为什么选你而不是别人" | 与竞品/替代方案的差异是否明确可视化 |
| 记忆点设计 | "看完几天还能想起核心信息" | 是否有金句、核心数据、标志性画面 |
| 结尾号召力 | "看完知道下一步做什么" | 结尾是否给出明确的行动号召 |

### 演示类审阅报告格式

```
## 演示类审阅报告

### 总体评价
- 开场吸引力：X/5
- 价值主张清晰度：X/5
- 场景共鸣度：X/5
- 差异性展示：X/5
- 记忆点设计：X/5
- 结尾号召力：X/5

### 黄金3页测试
前3页能否让目标受众产生"这个跟我有关，继续看"的意愿？

### 逐页审阅
...

### 改进优先级
1. [高] ...
```

## 审阅输出要求

1. 审阅报告至少包含：总体评价 + 关键页面审阅 + 改进优先级
2. 每个改进建议必须具体到页面和字段，不能说"建议优化"
3. 致命缺陷如果存在，必须排在改进优先级最高位
4. 审阅报告写入 `work/ppt/XXX/review-yyyyMMddHHmmss.md`
